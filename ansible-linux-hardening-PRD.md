## Project Goal

Generate a complete, production-quality Ansible project that automates **CIS Ubuntu 22.04 LTS Level 1 Benchmark** hardening on AWS EC2 instances, including Terraform infrastructure provisioning and automated compliance reporting in plain text and HTML.

This project serves as a public GitHub portfolio piece aligned to a **Senior Ansible Automation Engineer** CV. Every file must reflect real enterprise quality: idiomatic YAML, proper role structure, Galaxy-compatible metadata, and meaningful inline comments.

## Constraints & Non-Negotiables

- All Ansible YAML must pass `ansible-lint` with zero warnings
- All tasks must be **idempotent** (second run = zero changes)
- Terraform must use **t3.micro** instances (free-tier eligible on AWS)
- All AWS resources must carry these exact tags:
  - `fo:owner = sabatino`
  - `fo:platform = platform-engineering`
  - `fo:environment = sandbox`
  - `fo:purpose = poc-runbooks`
- Reports must be generated in **both** plain text (`.txt`) and **HTML** (`.html`)
- README must be in **English**, step-by-step, with explicit validation commands
- No placeholder comments — every comment must add context or explain the "why"

### `roles/cis_hardening/tasks/1_filesystem.yml`

CIS Section 1 — Filesystem Configuration.

Tasks (all use `ansible.builtin.lineinfile` on `/etc/modprobe.d/cis_filesystem.conf`):
- Disable: `cramfs`, `freevxfs`, `jffs2`, `hfs`, `hfsplus`, `udf` — each line: `install <module> /bin/true`
- Set login banner to `/etc/issue` and `/etc/issue.net` using `ansible.builtin.copy` with `content: "{{ login_banner }}"`
- Each task guarded by its corresponding `when: disable_<module>` variable
- Each task registers a result variable (e.g., `result_cramfs`)

Final task: `ansible.builtin.set_fact` to **append** to `compliance_results` a list of dicts with keys `id`, `title`, `status`. Status = `'PASS'` if result is not failed, `'FAIL'` otherwise.

CIS IDs to record: `CIS-1.1.1.1` through `CIS-1.1.1.6`, `CIS-1.7`.

### `roles/cis_hardening/tasks/3_network.yml`

CIS Section 3 — Network / sysctl hardening.

Use `ansible.posix.sysctl` module. All writes go to `/etc/sysctl.d/99-cis-hardening.conf`.

Settings to apply:
- `net.ipv4.ip_forward = 0` (when `disable_ip_forwarding`)
- `net.ipv4.conf.all.send_redirects = 0` AND `net.ipv4.conf.default.send_redirects = 0` (when `disable_send_redirects`)
- `net.ipv4.conf.all.accept_source_route = 0` AND `net.ipv4.conf.default.accept_source_route = 0` (when `disable_source_route_acceptance`)
- `net.ipv4.conf.all.accept_redirects = 0`, `net.ipv4.conf.default.accept_redirects = 0`, `net.ipv4.conf.all.secure_redirects = 0`, `net.ipv4.conf.default.secure_redirects = 0` (when `disable_icmp_redirects`)
- `net.ipv4.tcp_syncookies = 1` (when `enable_tcp_syn_cookies`)
- `net.ipv4.conf.all.rp_filter = 1` AND `net.ipv4.conf.default.rp_filter = 1` (when `enable_reverse_path_filtering`)
- `net.ipv6.conf.all.disable_ipv6 = 1` AND `net.ipv6.conf.default.disable_ipv6 = 1` (when `disable_ipv6`)

Record in `compliance_results`:
- `CIS-3.1`: IP forwarding disabled
- `CIS-3.2`: ICMP redirects disabled
- `CIS-3.2`: TCP SYN cookies enabled
- `CIS-3.2`: Reverse path filtering enabled

### `roles/cis_hardening/tasks/4_logging.yml`

CIS Section 4 — Logging & Auditing.

Tasks:
1. Install `auditd` with `ansible.builtin.package`
2. Enable and start `auditd` service (when `auditd_enabled`)
3. Write `/etc/audit/rules.d/cis_hardening.rules` using `ansible.builtin.copy` with this exact content:
   ```
   # CIS Benchmark audit rules
   # Time changes
   -a always,exit -F arch=b64 -S adjtimex -S settimeofday -k time-change
   -a always,exit -F arch=b32 -S adjtimex -S settimeofday -S stime -k time-change
   # User/group modifications
   -w /etc/group  -p wa -k identity
   -w /etc/passwd -p wa -k identity
   -w /etc/shadow -p wa -k identity
   -w /etc/sudoers -p wa -k identity
   # Network environment changes
   -a always,exit -F arch=b64 -S sethostname -S setdomainname -k system-locale
   # File deletion events
   -a always,exit -F arch=b64 -S unlink -S unlinkat -S rename -S renameat -F auid>=1000 -F auid!=4294967295 -k delete
   # Sudo usage
   -w /var/log/sudo.log -p wa -k actions
   # Kernel module loading
   -w /sbin/insmod  -p x -k modules
   -w /sbin/rmmod   -p x -k modules
   -w /sbin/modprobe -p x -k modules
   ```
   mode: `0640`, owner: root, group: root — notify `restart auditd`
4. Install `rsyslog` with `ansible.builtin.package`
5. Enable and start `rsyslog` (when `rsyslog_enabled`)
6. Set `$FileCreateMode 0640` in `/etc/rsyslog.conf` via `ansible.builtin.lineinfile` — notify `restart rsyslog`

Record in `compliance_results`:
- `CIS-4.1.1`: auditd installed and enabled
- `CIS-4.1.x`: Audit rules configured
- `CIS-4.2.1`: rsyslog installed and enabled
- `CIS-4.2.4`: rsyslog file permissions set

### `playbooks/audit.yml`

**Read-only** audit playbook. No changes applied. Safe to run in production.

Header comment must clearly state: `# Read-only audit playbook (no system changes)`.

Uses `ansible.builtin.shell` tasks with `changed_when: false` to check:
1. Unused filesystem modules — check `/etc/modprobe.d/*.conf` for `install <module> /bin/true`
2. sysctl parameters — compare actual vs expected for: `net.ipv4.ip_forward=0`, `net.ipv4.conf.all.send_redirects=0`, `net.ipv4.conf.all.accept_redirects=0`, `net.ipv4.tcp_syncookies=1`, `net.ipv4.conf.all.rp_filter=1`

4. auditd status — `systemctl is-active auditd`
5. rsyslog status — `systemctl is-active rsyslog`

Each shell task outputs lines prefixed `PASS:` or `FAIL:`.

After all checks:
- Combine all result lines into `all_audit_lines`
- Build `compliance_results` list of dicts (`id`, `title`, `status`)
- Compute `pass_count`, `fail_count`, `total_count`, `compliance_pct`
- Render both report templates (reuse same templates from the role)
- Fetch both reports to `reports/` with flat=true

## Quality Checklist

Before considering a file complete, verify:

- [ ] All YAML files start with `---`
- [ ] All task names follow the pattern `"SECTION | CIS-ID | Description"`
- [ ] All `ansible.builtin.*` FQCNs are used (not short module names)
- [ ] All `register:` variables are used or documented
- [ ] No `shell:` or `command:` tasks where a proper module exists
- [ ] `changed_when: false` on all shell tasks used only for checks
- [ ] `ignore_errors: true` only where intentional (e.g., missing services)
- [ ] All Jinja2 templates use `| default('N/A')` for potentially undefined vars
